Processor System

Your actors can now move! Assign movement components that update at different frequencies. Fast components update at 60 FPS, Slow components at 10 FPS. Watch the difference in motion.

Code Example: Processor-based Updates

// Processor-based Updates with Decorators
class MovementComponent extends Component {
  @RenderUpdate()  // 60 FPS - smooth movement
  fastUpdate(deltaTime: number) {
    const transform = this.actor.getComponent(TransformComponent);
    transform.x += this.vx * deltaTime;
    transform.y += this.vy * deltaTime;
  }

  @FixedTickUpdate()  // 10 FPS - game logic
  slowUpdate(deltaTime: number) {
    // AI decisions, collision detection, etc.
    this.updateAI(deltaTime);
  }
}

// Register and start processors
const renderProcessor = new RenderProcessor(); // 60 FPS
const gameLogicProcessor = new FixedTickProcessor("fixedTick", 10); // 10 FPS

ProcessorRegistry.register(renderProcessor);
ProcessorRegistry.register(gameLogicProcessor);

renderProcessor.start(); // Start the loops
gameLogicProcessor.start();

Query System

Now query your actor pool! Filter by components, colors, or movement types. Selected actors will be highlighted. Build complex queries to find exactly what you need.

Code Example: Scene Query System

// Scene Query System - Efficient Component Filtering
const scene = new Scene();

// Simple component queries
const animated = scene.query()
  .withComponent(AnimationComponent)
  .execute(scene);

const redActors = scene.query()
  .withComponent(RedComponent)
  .execute(scene);

// Complex multi-component queries
const redAndAnimated = scene.query()
  .withComponent(RedComponent, AnimationComponent)
  .limit(10)
  .execute(scene);

// Combining queries for complex logic
const fastMovingEnemies = scene.query()
  .withComponent(FastMovementComponent, EnemyComponent)
  .withoutTag('disabled')
  .orderBy((a, b) => a.getComponent(HealthComponent).health - b.getComponent(HealthComponent).health)
  .execute(scene);

// Process query results
animated.forEach(actor => {
  const render = actor.getComponent(RenderComponent);
  render.highlighted = true; // Visual feedback
});

Service Registry

Services provide shared functionality across your application using dependency injection. Components and systems resolve services by unique keys for decoupled architecture. Demonstrates service registration, resolution, and usage patterns.

Code Example: Service Registration & Dependency Injection

// Service Registration with Unique Keys
const IMouseServiceKey = Symbol.for("IMouseService");
const ICanvasServiceKey = Symbol.for("ICanvasService");

// Register services in the registry
serviceRegistry.register(IMouseServiceKey, new MouseService());
serviceRegistry.register(ICanvasServiceKey, new CanvasService());

// Service Usage in Components - Clean Dependency Injection
class AttractionComponent extends Component {
  @RenderUpdate()
  attract(deltaTime: number) {
    // Resolve services at runtime - fully decoupled
    const mouseService = journeyState.serviceRegistry
      .resolve<IMouseService>(IMouseServiceKey);
    
    if (!mouseService) return; // Graceful degradation
    
    const mousePos = mouseService.getPosition('step4-canvas');
    const transform = this.actor.getComponent(TransformComponent);
    
    // Service provides clean API for mouse interaction
    if (mouseService.isClicked()) {
      this.pulseMultiplier = 5; // Boost attraction on click
    }
    
    // Apply physics using service data
    const dx = mousePos.x - transform.x;
    const dy = mousePos.y - transform.y;
    const distance = Math.sqrt(dx * dx + dy * dy);
    
    if (distance > 5) {
      const force = this.attractionStrength / Math.max(distance, 50);
      transform.x += (dx / distance) * force * deltaTime;
      transform.y += (dy / distance) * force * deltaTime;
    }
  }
}

// Multiple services working together
class VisualEffectComponent extends Component {
  @RenderUpdate()
  createEffect(deltaTime: number) {
    const canvasService = this.resolveService<ICanvasService>(ICanvasServiceKey);
    const mouseService = this.resolveService<IMouseService>(IMouseServiceKey);
    
    // Services enable complex cross-system interactions
    if (canvasService && mouseService) {
      this.drawRippleEffect(canvasService, mouseService);
    }
  }
}

Complete System

Everything working together! Your actor pool with all components, processors running at different frequencies, queries filtering efficiently, and services handling interaction. This is the full power of the ACS.

Code Example: Complete ACS System Integration

// Complete ACS System Integration
class GameManager {
  constructor() {
    this.scene = new Scene('MainScene');
    this.serviceRegistry = new ServiceRegistry();
    this.processors = new Map();
  }

  async initialize() {
    // 1. Register all services
    this.serviceRegistry.register(IInputServiceKey, new InputService());
    this.serviceRegistry.register(IRendererServiceKey, new WebGPURenderer());
    this.serviceRegistry.register(IAudioServiceKey, new AudioService());
    
    // 2. Setup processors with different frequencies
    const renderProcessor = new RenderProcessor(); // 60 FPS
    const gameLogicProcessor = new FixedTickProcessor("gameLogic", 30); // 30 FPS
    const aiProcessor = new FixedTickProcessor("ai", 10); // 10 FPS
    
    ProcessorRegistry.register(renderProcessor);
    ProcessorRegistry.register(gameLogicProcessor);
    ProcessorRegistry.register(aiProcessor);
    
    // 3. Create game world with live filtering
    this.spawnGameEntities();
    
    // 4. Start all systems
    renderProcessor.start();
    gameLogicProcessor.start();
    aiProcessor.start();
    
    console.log('🚀 Complete ACS System Running');
  }

  // Live query filtering for performance
  getVisibleEnemies() {
    return this.scene.query()
      .withComponent(EnemyComponent, TransformComponent, RenderComponent)
      .withoutTag('hidden')
      .execute(this.scene);
  }
}
